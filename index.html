Project Manager to Frontend Developer

Project Manager to Frontend Developer: My Coding Story

I felt accomplished and challenged as a Project Manager. I was managing other professionals', I understood the ins-and-outs of my projects, I was anticipating risks and solving problems. I was a good at my job, but I was unsatisfied.

I had a nagging desire to create, design and 'do' something myself. Coding and app development had always been on my radar but I never seriously considered a career change into Tech. In June 2016 I took the opportunity of representing Australia at the Ultimate Frisbee World Championships in London to take a career break and extensively travel Eastern Europe.

*Quarter-life crisis in full swing*

While travelling, I entertained my curiosity in website, application and program development. In September 2016, I settled in Berlin and committed to learn the art and science of Web Development with the impression that I would teach myself to code.

A few months down the particularly bumpy road I haven't looked back. The 'self taught path' is flexible and liberating but is only as structured as your self control; there's no assignment or exam pressure (while isn't pleasurable, undoubtedly works), only your own motivation to make progress each day.

I made this website using HTML, CSS and javascript to track my progress and goals, practice the tools I learn and to share my 'Web Dev Notes' with the community. My vision is to create a collection of tutorials, examples and notes as a reference for myself and others learning Web Development. Please reach out! I would love to hear from you.

Website Layout Notes

  • CSS / HTML

The Display Property

Display property is the most important property for controlling layout. Every element has a default display value depending on what type of element it is. Display propery types include:

  • Block (or block level): Starts on a new line & stretches to the left and right as far as it can. I.e. div, p, form, header, footer, section.

  • Inline: Occupies only the space bound by the tags that define the inline element, they generally don't start a new line and contain only data or other inline elements i.e. span, a, img, strong, i.

  • None: Commonly used to hide & show elements without deleting and re-creating them

  • Other properties: There are many other properties all with unqiue uses, i.e. table, grid, flex...

Every element has a pre-defined display type, however, it can always be overridden to customise the display of elements i.e. making 'li' elements 'inline' for horizontal menus

Centering with 'Margin: auto'

  • Setting the width of an element prevents it from stretching to the edges of it's container
  • Setting left & right margins to 'auto' horizontally centers an element within it's container
  • The element takes up the width you specify, the remaining space either side is shared between the two margins
  • Using 'max-width' allows better handling of small windows, allowing a site to be usable on mobile

The Box Model widths biggest caveat

An element can appear larger than the width you set. An elements boarder & padding will stretch an element as seen with these 2 boxes

margin: 20px auto;

This was considered unintuitive so a css property called 'box-sizing' was created. When you set (box-sizing: border-box) on an element, the padding and border no londer increases its width

max-width: 500px;

Since box-sizing is a recent property, using the -webkit- and -moz- prefixes enables the feature in specific browsers. If you would like all elements to work this way, include the following CSS on your page:

* {

-webkit-box-sizing: border-box;

-moz-box-sizing: border-box;

box-sizing: border-box;

}

Position

Position specifies which type of positioning method is used for an element: Static, Relative, Absolute or Fixed.

  • Static

    Default value, a 'static' element is said to be 'not positioned'. Elements render in order as they appear in the document flow.

  • Relative

    Behaves the same as static unless you define further properties: (top, right, bottom, left) the element is then positioned relative to its normal position. i.e left:20px; will add 20 pixels to the elements left position

  • Fixed

    Positioned relative to the browser window - it stays in position when the page is scrolled. Commonly used for headers and footers. - Mobile browsers have poor support for fixed.

  • Absolute

    Absolute behaves like fixed except relative to the nearest positioned ancestor instead of the browser window. If an absolutely-positioned element has no positioned ancestors, it uses the document body, and still moves along with page scrolling. A "positioned" element is one whose position is anything except static.

Float

Originally for floating images inside blocks of text like I have done here, the float property has become one of the most commonly used tools for creating multiple column layouts on webpages

the element with the float set on it (the img element in this case) is taken out of the normal layout flow of the document and stuck to the left hand side of its parent container.

Any content that comes below the floated element in the normal layout flow will now wrap around it, filling up the space to the left hand side of it as far up as the top of the floated element. Note that floated content still obeys normal box behaviour with things such as margin and borders.
